## 📜 Code of Conduct

By participating in this project, you agree to be respectful and constructive. We welcome contributors of all backgrounds and skill levels.

---

## 🛠️ How to Contribute

There are many ways to contribute:

- 🐛 Report bugs
- 💡 Suggest new features
- 📝 Improve documentation
- 💻 Submit code improvements
- 🌍 Help with translations

---

## 🐛 Reporting Bugs

1. Check if the bug has already been reported in [Issues](../../issues)
2. If not, open a new issue with:
   - A clear title
   - Steps to reproduce the bug
   - Expected vs actual behavior
   - Screenshots if applicable

---

## 💡 Suggesting Features

1. Open a new issue with the label `enhancement`
2. Describe the feature and why it would be useful
3. Include mockups or examples if possible

## 🔀 Pull Request Process

```bash
# 1. Fork the repository
# Click "Fork" on GitHub

# 2. Clone your fork
git clone https://github.com/YOUR_USERNAME/studyswap.git
cd studyswap

# 3. Create a new branch
git checkout -b feature/your-feature-name

# 4. Make your changes and commit
git add .
git commit -m "Add: description of your changes"

# 5. Push to your fork
git push origin feature/your-feature-name

# 6. Open a Pull Request on GitHub
```

### Commit message format

```
Add: new feature description
Fix: bug description
Update: what was updated
Remove: what was removed
```

## 💻 Development Setup

```bash
# Install dependencies
npm install

# Run development server
npm run dev

# Check for linting errors
npm run lint
```

The app will be available at [http://localhost:3000](http://localhost:3000)
